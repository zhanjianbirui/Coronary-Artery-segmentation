# 本周工作报告

## 项目概述

冠状动脉 CTA 三维分割（ImageCAS 数据集，1000 例）。采用 **2.5D 方法**：将相邻切片堆叠为多通道输入，用 2D SegResNet 预测中心切片的血管掩码，推理后逐层拼回 3D 进行评估。

---

## 一、单方向 2.5D 基线搭建与优化

### 1.1 训练稳定性

初次训练使用 float16 AMP，前 19 轮正常（val\_dice 达到 0.79），第 20 轮 loss 突然全 nan 崩溃。

**原因诊断**：冠脉前景不到 1%，极稀疏场景下 float16 动态范围不够（最大 ~65504），梯度溢出。

**解决方案**（三管齐下）：
1. AMP 改为 **bfloat16**（A100 原生支持，动态范围与 fp32 相同）
2. 梯度级 nan/inf 检查：非有限时跳过 optimizer.step()
3. 学习率降至 3e-4

修复后全程零 nan，val\_dice 稳定达到 **0.8121**。

### 1.2 后处理参数调优

对 200 例全量测试集扫描 `min_voxels × max_gap` 组合：

| 配置 | Dice | clDice | Betti-0 err | HD95 |
|------|------|--------|-------------|------|
| mv=200, gap=15（初始） | 0.7945 | 0.8510 | 9.02 | 25.94 |
| **mv=300, gap=0（最优）** | **0.7955** | **0.8582** | **4.19** | **24.98** |

**关键发现**：端点重连（gap > 0）是**负优化**——直线重连连错的比连对的多，gap 越大 Betti-0 越差。关闭重连后 Betti-0 从 9.02 降至 4.19，纯收益。

### 1.3 阈值扫描

扫描 thr = 0.3 ~ 0.6，固定 mv=300, gap=0：高阈值 Dice 微升但 recall 和 HD95 变差。**结论：阈值保持 0.5**。

### 1.4 TTA（测试时增强）

面内 4-way 翻转 TTA（原图 + 水平翻 + 垂直翻 + 双翻），20 例子集验证：Dice +0.008，clDice +0.009，HD95 改善。**提升明显，值得采用。**

### 1.5 单方向当前最优结果（200 例全量测试集）

| Dice | clDice | Betti-0 err | HD95 |
|------|--------|-------------|------|
| 0.8027 | 0.8670 | 3.68 | 23.66 |

> 配置：SegResNet, k=2, crop=384, bf16, mv=300, gap=0, thr=0.5, TTA 开启

## 二、问题分析：单方向的局限

单方向（轴位）2.5D 的核心瓶颈在 **Betti-0 error = 3.68**，即预测的连通分量数与真实值平均相差约 3.68，反映血管存在断裂。

**根本原因**：轴位切片只在 axial 平面上有上下文。当血管走行近似平行于轴位面时（如前降支某些段），该方向的切片只能看到很小的截面，容易漏检或断裂。

后处理无法弥补：
- 端点重连（简化版和方向感知版）均被验证为负优化
- 小连通域去除只能减少碎片，无法桥接真正的断裂

**结论**：问题出在推理端信息不足，需要从模型/推理策略层面解决。

## 三、改进方案：三正交方向（Tri-planar）融合推理

### 3.1 核心思路

同一个模型（训练时已在三正交方向切片上混合训练），推理时沿 **轴位（axis=2）、冠状位（axis=1）、矢状位（axis=0）** 三个方向各推理一遍，得到三个概率体 p0, p1, p2，然后融合。

**物理直觉**：轴位看不清的血管，冠状位或矢状位可能正好垂直切过、信号清晰。三方向融合相当于"任一正交面看到血管就算数"，天然修复单方向的断裂盲区。

### 3.2 融合策略

两种融合方式对比：

| 方式 | 公式 | 预期特点 |
|------|------|----------|
| **mean** | (p0+p1+p2) / 3 | Dice/HD95 更友好，但会稀释"只在一个平面清晰"的信号 |
| **max** (noisy-OR) | max(p0, p1, p2) | 任一方向检出即保留，预期 clDice 更高、Betti-0 更低，代价是 FP 上升 |

### 3.3 实现（predict\_tri.py）

新增 `scripts/predict_tri.py`，两种运行模式：

**扫描模式**：对含难例（case 931/728/630）的 20 例子集，三轴各推一遍，交叉扫描 {mean, max} × {0.30, 0.35, 0.40, 0.45, 0.50} 共 10 种组合，输出对比表。

```bash
PYTHONPATH=. python scripts/predict_tri.py \
    --cache-dir /path/to/cache \
    --ckpt runs/exp_tri/best.pth \
    --sweep --case-ids 931 728 630 --extra-cases 17 \
    --thr-grid 0.30 0.35 0.40 0.45 0.50
```

**全量模式**：选定最优组合后，跑全部 200 例测试集，输出与单轴 baseline 口径一致的指标。

```bash
PYTHONPATH=. python scripts/predict_tri.py \
    --cache-dir /path/to/cache \
    --ckpt runs/exp_tri/best.pth \
    --fixed-fuse max --thr 0.40 \
    --min-voxels 300 --max-gap 0
```

### 3.4 评估关注点

1. **max vs mean 的相对趋势**：如果 max 的 clDice 明显高、Betti-0 明显低，说明"任一正交面看到就算数"确实在桥接断裂
2. **难例逐 case 分析**：931/728/630 是已知的真断裂 case，直接盯它们的 Betti-0 error 在 max 融合下有没有下降
3. **不和单轴绝对值直接比**：子集小、raw 无后处理、专挑难例，数值天然偏难；看的是 mean/max 的相对关系
